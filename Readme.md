### 1. 核心技术栈
### 整体架构
```plaintext
Electron 应用
├─ 主进程（Main Process）
│  ├─ MQTT Broker（aedes）：监听 1883 端口，处理手表连接与消息路由
│  ├─ 协议解析服务：解析手表上行 16 进制数据，生成下行应答帧
│  ├─ 设备管理：维护 300 台手表的连接状态、MAC 映射、数据缓存
│  └─ 日志服务：记录连接、数据、错误日志，支持导出
└─ 渲染进程（Renderer Process）
├─ 设备监控界面：显示在线设备数、每台设备的健康数据（心率/血氧/温度）
├─ MQTT 配置界面：修改端口、QoS 等级、并发限制
├─ 数据日志界面：实时展示上行/下行数据帧，支持筛选与搜索
└─ 系统设置：WiFi 局域网检测、服务启停控制
```
